# Участие в разработке Taidon

Спасибо за интерес к проекту!  
Taidon развивается открытым сообществом, включая студентов, волонтёров и исследователей.  
Мы стараемся держать процесс простым, чтобы вход был лёгким.

---

## Как внести вклад

### 1. Найдите задачу

Посмотрите трекер задач. Хорошие стартовые варианты:

- `good first issue` — простые задачи для новичков  
- `help wanted` — задачи, которым нужна помощь  

Если хотите предложить собственную идею, смело открывайте новый issue.

---

### 2. Создайте ветку

Пожалуйста, не коммитьте напрямую в `main`.

Используйте соглашение по именованию веток:

- Фичи: `feature/<short-name>`
- Багфиксы: `fix/<short-name>`
- Документация: `docs/<short-name>`

Пример:

```bash
git checkout -b feature/query-snapshots
```

---

### 3. Модель разработки

Taidon — многоязычный монорепозиторий.  
У каждого подпроекта (`frontend/*`, `backend/*`, `docs`, `research`) своя настройка, инструменты и README.

Перед пушем изменений:

- следуйте tooling‑инструкциям в README соответствующего модуля  
- запускайте его тесты и линтеры  
- убедитесь, что не ломаете другие модули  

---

### 4. Pull Requests

При открытии PR:

- опишите, что делает изменение  
- приложите скриншоты/логи при необходимости  
- укажите связанные issue  
- держите историю коммитов разумно чистой  

Правила ревью:

- каждый PR должен быть одобрен хотя бы одним ревьюером  
- CI должен проходить  

Не мержите собственный PR без явного разрешения.

---
